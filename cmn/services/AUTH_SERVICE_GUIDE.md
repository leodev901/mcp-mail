# AUTH_SERVICE_GUIDE.md

## 목적
- `cmn/services/auth_service.py` 의 `/api/auth` 토큰 발급 흐름을 빠르게 이해하기 위한 가이드입니다.

## 문제 정의
- 캐시 확인, lock, DB 조회, 외부 API 호출을 한 함수에 모두 적으면 `if/else` 중첩이 깊어져 읽기 어려워집니다.

## 접근 방법
- `get_auth_token()` 은 흐름만 보이게 유지합니다.
- 세부 작업은 작은 헬퍼 함수로 분리합니다.

## 현재 흐름
1. 입력값을 정규화합니다.
2. 메모리 캐시를 먼저 확인합니다.
3. cache miss 면 key 단위 lock 을 잡습니다.
4. lock 안에서 캐시를 다시 확인합니다.
5. 그래도 없으면 DB 에서 Graph 설정을 읽습니다.
6. DB 세션을 닫은 뒤 외부 토큰 API 를 호출합니다.
7. 발급된 토큰을 캐시에 저장합니다.

## 왜 이렇게 했는지
- early return 으로 분기 중첩을 줄여 읽기 쉽게 만들기 위해서입니다.
- DB 세션 사용 구간과 외부 API 호출 구간을 분리해 커넥션 점유 시간을 줄이기 위해서입니다.

## 한 줄 요약
- `AuthService` 는 흐름 제어만 읽히게 하고, 세부 작업은 헬퍼 함수로 분리하면 분기 중첩을 깔끔하게 줄일 수 있습니다.
